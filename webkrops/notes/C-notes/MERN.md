## 1. MongoDB Questions

---

### Q1. What is MongoDB? How is it different from SQL databases?

**Answer:**
MongoDB is a **NoSQL database** that stores data as documents in JSON-like format (called BSON). Unlike SQL databases (like MySQL), it does not use tables and rows.

| Feature      | MongoDB (NoSQL)            | MySQL (SQL)             |
| ------------ | -------------------------- | ----------------------- |
| Data storage | Documents (JSON)           | Tables (rows & columns) |
| Schema       | Flexible (no fixed schema) | Fixed schema            |
| Joins        | Using `$lookup`          | Native JOIN support     |
| Scaling      | Horizontal (easy)          | Vertical (harder)       |

**When to use MongoDB:** When your data structure changes often, or you need to store nested/complex data.

---

### Q2. What is a Document and a Collection in MongoDB?

**Answer:**

- **Document** = One record (like a row in SQL), stored as JSON/BSON
- **Collection** = A group of documents (like a table in SQL)

```json
// Example Document
{
  "_id": "64abc123",
  "name": "Rahul",
  "email": "rahul@gmail.com",
  "age": 22
}
```

---

### Q3. What is Mongoose? Why do we use it?

**Answer:**
Mongoose is an **ODM (Object Data Modeling)** library for MongoDB and Node.js.Mongoose is an ODM library for MongoDB that makes it easier to store, validate, and manage data in Node.js applications.

**Why use it?**

- Adds schema/structure to MongoDB (which is schema-less)
- Makes validation easy
- Provides helper methods like `.find()`, `.save()`, `.populate()`

```javascript
const mongoose = require('mongoose');

const userSchema = new mongoose.Schema({
  name: { type: String, required: true },
  email: { type: String, unique: true },
  age: Number
});

const User = mongoose.model('User', userSchema);
```

---

### Q4. What are the basic CRUD operations in MongoDB/Mongoose?

**Answer:**

```javascript
// CREATE
const user = new User({ name: 'Priya', email: 'priya@gmail.com' });
await user.save();

// READ
const allUsers = await User.find();
const oneUser = await User.findById(id);

// UPDATE
await User.findByIdAndUpdate(id, { name: 'Updated Name' }, { new: true });

// DELETE
await User.findByIdAndDelete(id);
```

### Q5. What is the difference between `find()` and `findOne()`?

**Answer:**

- `find()` — returns **all** documents matching the query (as an array)
- `findOne()` — returns **only the first** matching document (as an object)

```javascript
User.find({ age: 22 })     // Returns array of all users with age 22
User.findOne({ age: 22 })  // Returns only first user with age 22
```

---

### Q6. What is `_id` in MongoDB? Is it auto-generated?

**Answer:**
`_id` is the **primary key** in MongoDB. Yes, it is auto-generated by MongoDB as an `ObjectId` if you don't provide one. It is a 12-byte unique identifier.

```javascript
// ObjectId looks like:
"64abc1234567890abc123456"
```

---

### Q7. What is the `populate()` method in Mongoose?

**Answer:**
`populate()` is used to **replace a reference (ObjectId) with the actual document data** from another collection — similar to a JOIN in SQL.

```javascript
// Post schema references User
const postSchema = new mongoose.Schema({
  title: String,
  author: { type: mongoose.Schema.Types.ObjectId, ref: 'User' }
});

// Without populate — author is just an ID
const post = await Post.findById(id);
// { title: "My Post", author: "64abc123..." }

// With populate — author is full user object
const post = await Post.findById(id).populate('author', 'name email');
// { title: "My Post", author: { name: "Rahul", email: "rahul@gmail.com" } }
```

---

### Q8. What are indexes in MongoDB?

**Answer:**
Indexes improve the **speed of queries**. Without an index, MongoDB scans every document (called a **collection scan**). With an index, it can jump directly to matching documents.

```javascript
// Create an index on email field
userSchema.index({ email: 1 });  // 1 = ascending, -1 = descending

// Unique index — no two documents can have the same email
email: { type: String, unique: true }
```

## 2. Express.js Questions

---

### Q9. What is Express.js?

**Answer:**
Express is a **minimal, fast web framework for Node.js**. It makes building REST APIs and web servers easier by providing features like routing, middleware, and request/response handling.

```javascript
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(5000, () => console.log('Server running on port 5000'));
```

---

### Q10. What is Middleware in Express?

**Answer:**
Middleware is a **function that runs between the request and response**. It has access to `req`, `res`, and `next`. Call `next()` to move to the next middleware.

```javascript
// Custom logging middleware
app.use((req, res, next) => {
  console.log(`${req.method} ${req.path}`);
  next(); // Move to next middleware or route
});

// Built-in middleware
app.use(express.json()); // Parse JSON body
app.use(express.urlencoded({ extended: true })); // Parse form data
```

**Common uses:** Logging, authentication, body parsing, error handling.

---

### Q11. What is the difference between `app.use()` and `app.get()`?

**Answer:**

- `app.use()` — runs for **all HTTP methods** (GET, POST, PUT, DELETE)
- `app.get()` — runs only for **GET requests**

```javascript
app.use('/api', middleware);   // Matches any method at /api
app.get('/users', handler);    // Only GET /users
app.post('/users', handler);   // Only POST /users
```

---

### Q12. What is a Router in Express?

**Answer:**
`express.Router()` lets you create **modular, mini-applications** to organize routes. Keeps your code clean instead of putting all routes in one file.

```javascript
// routes/user.routes.js
const router = require('express').Router();

router.get('/', getAllUsers);
router.get('/:id', getUserById);
router.post('/', createUser);

module.exports = router;

// server.js
app.use('/api/users', require('./routes/user.routes'));
```

---

### Q13. How do you handle errors in Express?

**Answer:**
Error handling middleware has **4 parameters**: `(err, req, res, next)`. Place it after all routes.

```javascript
// In your route
app.get('/user/:id', async (req, res, next) => {
  try {
    const user = await User.findById(req.params.id);
    if (!user) return res.status(404).json({ message: 'User not found' });
    res.json(user);
  } catch (err) {
    next(err); // Pass error to error handler
  }
});

// Error handling middleware (place at the end)
app.use((err, req, res, next) => {
  console.error(err.message);
  res.status(500).json({ message: 'Something went wrong' });
});
```

---

### Q14. What is the difference between `req.params`, `req.query`, and `req.body`?

**Answer:**

|                | What it is                   | Example URL                      | How to access                    |
| -------------- | ---------------------------- | -------------------------------- | -------------------------------- |
| `req.params` | URL path parameters          | `/users/:id` → `/users/123` | `req.params.id` → `"123"`   |
| `req.query`  | URL query string             | `/users?age=22`                | `req.query.age` → `"22"`    |
| `req.body`   | POST/PUT request body (JSON) | `{ "name": "Rahul" }`          | `req.body.name` → `"Rahul"` |

---

### Q15. What is CORS? How do you enable it in Express?

**Answer:**
**CORS (Cross-Origin Resource Sharing)** is a browser security feature that blocks requests from a different domain. For example, your React app on `localhost:3000` cannot call your Express API on `localhost:5000` without enabling CORS.

```javascript
// Install: npm install cors
const cors = require('cors');

app.use(cors()); // Allow all origins (for development)

// For production - be specific
app.use(cors({ origin: 'https://yourapp.com' }));
```

## 3. React.js Questions

---

### Q16. What is React? What problem does it solve?

**Answer:**
React is a **JavaScript library for building user interfaces**, created by Facebook. It solves the problem of efficiently updating the UI when data changes, using a **Virtual DOM** instead of manipulating the real DOM directly.

---

### Q17. What is JSX?

**Answer:**
JSX is **JavaScript + XML/HTML syntax** — it lets you write HTML inside JavaScript. Browsers can't read JSX directly; it gets compiled to regular JavaScript by Babel.

```jsx
// JSX
const element = <h1>Hello, {name}!</h1>;

// What it compiles to (you don't write this)
const element = React.createElement('h1', null, `Hello, ${name}!`);
```

---

### Q18. What is the difference between a Functional Component and Class Component?

**Answer:**
Modern React uses **functional components**. Class components are older and rarely used in new projects.

```jsx
// Functional Component (modern - use this)
function Greeting({ name }) {
  return <h1>Hello, {name}!</h1>;
}

// Class Component (older style)
class Greeting extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}!</h1>;
  }
}
```

---

### Q19. What is State in React? How do you use `useState`?

**Answer:**
State is **data that belongs to a component** and can change over time. When state changes, React re-renders the component.

```jsx
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0); // Initial value = 0

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increase</button>
    </div>
  );
}
```

---

### Q20. What are Props in React?

**Answer:**
Props (properties) are **read-only inputs passed from parent to child** components. A child cannot change its own props.

```jsx
// Parent component
function App() {
  return <UserCard name="Rahul" age={22} />;
}

// Child component
function UserCard({ name, age }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Age: {age}</p>
    </div>
  );
}
```

---

### Q21. What is `useEffect`? When does it run?

**Answer:**
`useEffect` lets you perform **side effects** in functional components (like fetching data, subscriptions, DOM manipulation).

```jsx
import { useState, useEffect } from 'react';

function Users() {
  const [users, setUsers] = useState([]);

  // Runs once after component mounts (empty dependency array)
  useEffect(() => {
    fetch('/api/users')
      .then(res => res.json())
      .then(data => setUsers(data));
  }, []); // [] = run only on mount

  return <ul>{users.map(u => <li key={u._id}>{u.name}</li>)}</ul>;
}
```

**When does useEffect run?**

- `useEffect(() => {...})` — after every render
- `useEffect(() => {...}, [])` — only once on mount
- `useEffect(() => {...}, [value])` — when `value` changes

---

### Q22. What is the Virtual DOM?

**Answer:**
The Virtual DOM is a **lightweight copy of the real DOM kept in memory**. When state changes, React:

1. Creates a new virtual DOM tree
2. Compares it with the old one (called **diffing**)
3. Updates **only the changed parts** in the real DOM (called **reconciliation**)

This makes React fast because real DOM updates are expensive.

---

### Q23. What is the `key` prop in React lists?

**Answer:**
The `key` prop helps React **identify which items in a list have changed**. Always use a unique, stable value (like `_id` from database) — not the array index.

```jsx
// Good - using unique ID
{users.map(user => (
  <UserCard key={user._id} user={user} />
))}

// Bad - using index (can cause bugs in dynamic lists)
{users.map((user, index) => (
  <UserCard key={index} user={user} />
))}
```

---

### Q24. What is conditional rendering in React?

**Answer:**
Showing or hiding components based on a condition.

```jsx
function Dashboard({ isLoggedIn }) {
  return (
    <div>
      {isLoggedIn ? (
        <h1>Welcome back!</h1>
      ) : (
        <h1>Please log in</h1>
      )}

      {/* Short circuit - only renders if isLoggedIn is true */}
      {isLoggedIn && <UserProfile />}
    </div>
  );
}
```

---

### Q25. How do you make an API call in React?

**Answer:**
Use `fetch` or `axios` inside `useEffect`.

```jsx
import { useState, useEffect } from 'react';
import axios from 'axios';

function Products() {
  const [products, setProducts] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    axios.get('/api/products')
      .then(res => {
        setProducts(res.data);
        setLoading(false);
      })
      .catch(err => console.error(err));
  }, []);

  if (loading) return <p>Loading...</p>;

  return (
    <ul>
      {products.map(p => <li key={p._id}>{p.name}</li>)}
    </ul>
  );
}
```

### Q26. What is React Router? How do you use it?

**Answer:**
React Router is a library for **client-side navigation** in React apps — moving between pages without full page reload.

```jsx
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">Home</Link>
        <Link to="/about">About</Link>
      </nav>

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/users/:id" element={<UserDetail />} />
        <Route path="*" element={<NotFound />} />
      </Routes>
    </BrowserRouter>
  );
}
```

## 4. Node.js Questions

---

### Q27. What is Node.js? Why is it used for backend?

**Answer:**
Node.js is a **JavaScript runtime built on Chrome's V8 engine** — it lets you run JavaScript outside the browser (on the server).

**Why use it:**

- Same language (JavaScript) on frontend and backend
- Non-blocking, asynchronous — handles many requests efficiently
- Large npm ecosystem
- Fast for I/O-heavy operations (APIs, real-time apps)

---

### Q28. What does "non-blocking" or "asynchronous" mean in Node.js?

**Answer:**
Node.js doesn't wait for one task to finish before starting the next. It starts a task, moves on, and comes back when the result is ready.

```javascript
// Blocking (BAD in Node)
const data = readFileSync('file.txt'); // Waits here
console.log(data);
console.log('This runs after');

// Non-blocking (GOOD)
readFile('file.txt', (err, data) => {
  console.log(data); // Runs when ready
});
console.log('This runs immediately, without waiting');
```

### Q31. What is `dotenv`? Why is it important?

**Answer:**
`dotenv` loads environment variables from a `.env` file. Used to store **sensitive config** (database URLs, API keys, JWT secrets) out of the code.


### Q32. What is `async/await`? How is it different from callbacks?

**Answer:**
`async/await` is modern syntax to write **asynchronous code that looks synchronous** — easier to read and debug than callbacks or `.then()` chains.

```javascript
// Callback style (older, harder to read)
User.findById(id, function(err, user) {
  if (err) return console.error(err);
  Post.find({ author: user._id }, function(err, posts) {
    // "Callback hell" - deeply nested
  });
});

// Async/Await style (modern, clean)
async function getUserWithPosts(id) {
  try {
    const user = await User.findById(id);
    const posts = await Post.find({ author: user._id });
    return { user, posts };
  } catch (err) {
    console.error(err);
  }
}
```

## 5. JavaScript Core (Must Know)

---

### Q33. What is the difference between `var`, `let`, and `const`?

**Answer:**

|              | `var`            | `let`               | `const`             |
| ------------ | ------------------ | --------------------- | --------------------- |
| Scope        | Function scope     | Block scope           | Block scope           |
| Reassignable | Yes                | Yes                   | No                    |
| Hoisting     | Yes (as undefined) | Yes (not initialized) | Yes (not initialized) |

```javascript
var x = 1;   // Avoid in modern JS
let y = 2;   // Use when value will change
const z = 3; // Use when value won't change (preferred)
```

---

### Q34. What is the difference between `==` and `===`?

**Answer:**

- `==` — compares values **with type conversion** (loose equality)
- `===` — compares values **without type conversion** (strict equality)

```javascript
'5' == 5    // true  (string converted to number)
'5' === 5   // false (different types)
null == undefined  // true
null === undefined // false

// Always use === in real code
```

---

### Q35. What is a Promise in JavaScript?

**Answer:**
A Promise is an object that represents a **future value** — something that will be available later (success or failure).

```javascript
// States: pending → fulfilled OR rejected
const promise = new Promise((resolve, reject) => {
  const success = true;
  if (success) resolve('Data loaded!');
  else reject('Something went wrong');
});

promise
  .then(data => console.log(data))   // On success
  .catch(err => console.error(err)); // On failure
```

---

### Q36. What is the spread operator (`...`)?

**Answer:**
The spread operator **expands** an array or object into individual elements.

```javascript
// Arrays
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5]

// Objects (common in React state updates)
const user = { name: 'Rahul', age: 22 };
const updatedUser = { ...user, age: 23 }; // { name: 'Rahul', age: 23 }

// Function parameters
function sum(...numbers) {
  return numbers.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3, 4); // 10
```

---

### Q37. What is destructuring in JavaScript?

**Answer:**
Destructuring lets you **unpack values from arrays or objects** into variables.

```javascript
// Object destructuring
const user = { name: 'Priya', age: 22, city: 'Hyderabad' };
const { name, age } = user;
console.log(name); // 'Priya'

// Array destructuring
const [first, second] = [10, 20, 30];
console.log(first); // 10

// Very common in React
const [count, setCount] = useState(0);
```

---

### Q38. What is `localStorage`? When would you use it?

**Answer:**
`localStorage` stores **key-value pairs in the browser** that persist even after the browser is closed (unlike sessionStorage).

```javascript
// Common use: storing JWT token
localStorage.setItem('token', 'eyJhbGciOiJIUzI1...');
const token = localStorage.getItem('token');
localStorage.removeItem('token'); // On logout
```

## 6. REST API & HTTP Concepts

---

### Q39. What is a REST API?

**Answer:**
REST (Representational State Transfer) is a set of **conventions for building APIs** using HTTP methods. A REST API lets the frontend and backend communicate.

| HTTP Method | Action           | Example URL               |
| ----------- | ---------------- | ------------------------- |
| GET         | Read data        | `GET /api/users`        |
| POST        | Create data      | `POST /api/users`       |
| PUT         | Update (full)    | `PUT /api/users/:id`    |
| PATCH       | Update (partial) | `PATCH /api/users/:id`  |
| DELETE      | Delete data      | `DELETE /api/users/:id` |

---

### Q40. What are HTTP status codes? Name the common ones.

**Answer:**

| Code | Meaning               | When to use                 |
| ---- | --------------------- | --------------------------- |
| 200  | OK                    | Successful GET/PUT          |
| 201  | Created               | Successful POST             |
| 400  | Bad Request           | Invalid input from client   |
| 401  | Unauthorized          | Not logged in               |
| 403  | Forbidden             | Logged in but no permission |
| 404  | Not Found             | Resource doesn't exist      |
| 500  | Internal Server Error | Bug in your server code     |

---

### Q41. What is the difference between `GET` and `POST`?

**Answer:**

- `GET` — fetches data, **no request body**, parameters in URL, can be cached/bookmarked
- `POST` — sends data to server, **has request body**, not cached, used for creating data

```javascript
// GET - data in URL
fetch('/api/users?page=1')

// POST - data in body
fetch('/api/users', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name: 'Rahul', email: 'r@gmail.com' })
});
```

---

### Q42. What is JWT? How does it work in a MERN app?

**Answer:**
JWT (JSON Web Token) is used for **authentication**. It is a token the server gives you after login, which you send with every future request to prove who you are.

```
JWT Structure:
Header.Payload.Signature
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VySWQiOiIxMjMifQ.ABCDEF
```

**Flow in MERN:**

1. User logs in → Server verifies credentials → Creates JWT → Sends to client
2. Client stores JWT in localStorage
3. For every protected request, client sends JWT in header
4. Server verifies the token before sending data

```javascript
// Creating token (backend)
const jwt = require('jsonwebtoken');
const token = jwt.sign({ userId: user._id }, process.env.JWT_SECRET, { expiresIn: '7d' });

// Sending token (frontend)
axios.get('/api/profile', {
  headers: { Authorization: `Bearer ${token}` }
});
```

**Answer:**

- **Git** — version control system installed on your computer to track code changes
- **GitHub** — cloud platform to host Git repositories and collaborate with others

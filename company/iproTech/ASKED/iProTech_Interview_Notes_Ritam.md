## 3. How do you differentiate users in a chat application?

### Interview-Ready Answer

Each user has a unique identifier (User ID). Messages are stored with the user's ID, and authentication mechanisms such as JWT help identify the user. When a new message arrives, I fetch the user's previous conversation history and provide the relevant context to the AI so that responses remain personalized and separated from other users.

### Key Terms
- User ID
- JWT Authentication
- Session ID
- Chat History
- Context Management

## 4. What is MCP?

### Interview-Ready Answer

MCP (Model Context Protocol) is a standard protocol that allows AI models to communicate with external tools, APIs, databases, and services through a common interface. Instead of building custom integrations for every tool, MCP provides a standardized way for AI agents to discover and use available capabilities.

### Example

If an AI assistant needs to create an Instagram post, search the web, read files, or access a database, it can use an MCP server. The server performs the action and returns the result back to the AI model.

### Key Terms
- MCP
- Tools
- APIs
- External Services
- Agent Integration

## 5. Difference Between MongoDB and Pinecone

### Interview-Ready Answer

MongoDB is a document database that stores data as BSON (Binary JSON) documents. It is used for application data such as users, chats, products, and orders.

Pinecone is a vector database that stores embeddings generated by AI models. It is mainly used for semantic search, similarity search, and RAG applications.

In AI applications, I usually store the original content in MongoDB and the embeddings in Pinecone.

### MongoDB
- Stores BSON documents
- Good for application data
- Query using fields and indexes

### Pinecone
- Stores vector embeddings
- Good for semantic search
- Query using similarity search

---

## 6. What are Embeddings?

### Interview-Ready Answer

Embeddings are numerical vector representations of text, images, or other data generated by embedding models. These vectors capture semantic meaning, allowing similar content to be found through vector similarity search.

### Example

Text:
"I love JavaScript"

Embedding:
[0.12, -0.45, 0.88, 0.34, ...]

These vectors are stored in vector databases such as Pinecone.
